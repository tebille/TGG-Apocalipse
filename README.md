# TGG-Apocalipse 사용 설명서

마인크래프트 안에서 러스트와 비슷한 파밍, 기지 건축, 유지비, 레이드 시스템을 운영하는 방법을 설명합니다.

이 설명서는 다음 순서로 읽으면 쉽습니다.

1. 서버 관리자는 `빠른 시작`을 따라 최초 설정
2. 파밍 노드와 관리자 GUI 설정
3. 일반 유저에게 `플레이 방법` 안내
4. 문제가 생기면 마지막 `문제 해결` 확인

---

## 1. 필요한 서버 환경

- Java 21
- Spigot 1.21.1
- WorldGuard 7.0.17
- WorldEdit: WorldGuard가 요구하는 호환 버전
- ProtocolLib는 현재 필수가 아닙니다.
---

## 2. 빠른 시작

### 2-1. 플러그인 설치

1. 서버를 종료합니다.
2. WorldEdit과 WorldGuard를 먼저 설치합니다.
3. TGG-Apocalipse Shadow JAR를 `plugins` 폴더에 넣습니다.
4. 서버를 실행합니다.
5. `plugins/TGG-Apocalipse/config.yml`이 생성됐는지 확인합니다.

플러그인의 게임 데이터는 같은 폴더의 `apocalipse.db`에 자동 저장됩니다.

### 2-2. 메인 게임 구역 만들기

WorldEdit 도끼로 두 지점을 선택하고 WorldGuard 구역을 만듭니다.

```text
//wand
/rg define apocalypse
```

`config.yml`에서 메인 구역 ID를 맞춥니다.

```yaml
gameRegions:
  - apocalypse
```

일반 유저의 건축, 도구함, 파밍 상호작용은 이 게임 구역을 기준으로 처리됩니다.

### 2-3. 파밍 구역 만들기

광산, 폐허, 도로처럼 자원과 상자를 배치할 구역을 별도로 지정합니다.

```text
//wand
/rg define farm_1
```

`config.yml`에 구역 ID를 추가합니다.

```yaml
farming:
  worldGuardRegions:
    - farm_1
```

파밍 구역에서는 일반 유저가 블럭을 설치하거나 파괴할 수 없습니다. 파밍지 판정은 파밍 블럭 개수가 아니라 WorldGuard 구역으로 결정됩니다.

설정을 수정한 뒤 적용합니다.

```text
/apoc reload
```

## 3. 관리자 권한과 기본 명령어

관리자 명령어 권한은 `apocalipse.admin`이며 기본적으로 OP에게 주어집니다. `/apocalypse` 대신 짧은 별칭 `/apoc`을 사용할 수 있습니다.

### 아이템 지급

기본 형식은 다음과 같습니다.

```text
/apoc give <플레이어> <아이템 ID> [수량]
```

| 아이템 ID | 설명 |
|---|---|
| `cupboard` | 도구함 |
| `hammer` | 건축 망치 |
| `grinder` | 갈갈이 |
| `wood` | 나무 건축 블럭 |
| `stone` | 돌 건축 블럭 |
| `metal` | 철 건축 블럭 |
| `wood_door` | 나무 문 |
| `sheet_door` | 판금 문 |
| `steel_door` | 강철 문 |
| `sleeping_bag` | 침낭 |
| `satchel` | 가방 폭탄 |
| `c4` | C4 |
| `rocket_launcher` | 로켓 런처 |
| `rocket` | 로켓 탄약 |

예시:

```text
/apoc give Steve cupboard
/apoc give Steve wood 64
/apoc give Steve rocket_launcher
/apoc give Steve rocket 16
```

아이템 제작은 별도 제작 플러그인에서 연결하면 됩니다.

### 전체 명령어

| 명령어 | 기능 |
|---|---|
| `/apoc help` | 명령어 도움말 |
| `/apoc reload` | 설정 다시 불러오기 |
| `/apoc admin panel` | 관리자 통합 GUI 열기 |
| `/apoc admin build on` | 강제 건축 모드 켜기 |
| `/apoc admin build off` | 강제 건축 모드 끄기 |
| `/apoc admin build toggle` | 강제 건축 모드 전환 |
| `/apoc admin loot <등급>` | 상자 보상 설정 GUI |
| `/apoc admin grinder <번호>` | 갈갈이 레시피 설정 GUI |
| `/apoc node resource <종류>` | 바라보는 블럭을 자원 노드로 등록 |
| `/apoc node loot <등급> can` | 바라보는 블럭을 깡통 노드로 등록 |
| `/apoc node loot <등급> crate` | 바라보는 블럭을 상자 노드로 등록 |
| `/apoc node grinder` | 바라보는 블럭을 갈갈이로 등록 |

명령어에는 탭 자동완성이 적용됩니다.

## 4. 파밍지 설정 방법

### 4-1. 자원 블럭 등록

1. `config.yml`의 `farming.resources`에 자원 종류를 만듭니다.
2. 파밍 구역 안에 실제 블럭을 설치합니다.
3. 해당 블럭을 6블럭 이내에서 바라봅니다.
4. 등록 명령어를 실행합니다.

```text
/apoc node resource WOOD
/apoc node resource STONE
/apoc node resource METAL
```

기본 자원 종류는 다음과 같습니다.

- `WOOD`: 도끼 계열 사용
- `STONE`: 곡괭이 계열 사용
- `METAL`: 곡괭이 계열 사용

자원마다 필요한 타격 횟수, 리젠 시간, 드롭 아이템을 설정할 수 있습니다.

### 4-2. 깡통 등록

깡통으로 사용할 블럭을 바라보고 실행합니다.

```text
/apoc node loot COMMON can
```

깡통은 설정된 근접 무기로 좌클릭해야 파밍됩니다.

### 4-3. 파밍 상자 등록

상자로 사용할 블럭을 바라보고 실행합니다.

```text
/apoc node loot COMMON crate
/apoc node loot RARE crate
/apoc node loot ELITE crate
```

상자는 우클릭하면 보상 GUI가 열립니다. 남은 보상은 GUI를 닫을 때 플레이어에게 반환됩니다.

### 4-4. 상자 보상 설정

간단한 방법:

```text
/apoc admin panel
```

또는 등급을 바로 엽니다.

```text
/apoc admin loot COMMON
```

GUI 위쪽 5줄에 보상 아이템을 넣습니다. 아이템 종류와 스택 수량이 그대로 저장됩니다. GUI를 닫으면 자동 저장됩니다.

상자 등급별 최소·최대 보상 추첨 횟수는 `loot.tiers`에서 설정합니다.

## 5. 갈갈이 설정과 사용

### 5-1. 갈갈이 설치

갈갈이는 기지에 설치하는 시설이 아닙니다. 관리자가 파밍 구역에만 설치합니다.

```text
/apoc admin build on
/apoc give <관리자> grinder
```

파밍 WorldGuard 구역에 갈갈이를 설치하면 자동으로 갈갈이 노드에 등록됩니다. 설치가 끝나면 강제 건축 모드를 끕니다.

```text
/apoc admin build off
```

일반 유저와 강제 건축 모드가 꺼진 관리자는 갈갈이를 설치할 수 없습니다.

### 5-2. 갈갈이 레시피 만들기

```text
/apoc admin grinder 1
```

GUI에서 다음처럼 설정합니다.

- 13번 칸: 분해할 입력 아이템
- 28~34번 칸: 분해 후 나올 결과 아이템
- 입력 아이템의 스택 수량: 한 번 처리할 때 소모되는 수량
- 결과 아이템의 스택 수량: 한 번 처리할 때 지급되는 수량

GUI를 닫으면 SQLite에 자동 저장됩니다.

### 5-3. 일반 유저 사용법

1. 갈갈이를 우클릭합니다.
2. 위쪽 입력 구역에 분해할 아이템을 넣습니다.
3. `시작` 버튼을 누릅니다.
4. 결과 아이템은 출력 구역에 쌓입니다.
5. 입력 아이템이 모두 사라지면 자동 종료됩니다.
6. GUI를 닫으면 남은 입력·출력 아이템이 플레이어에게 반환됩니다.

## 6. 도구함과 기지 건축

### 6-1. 도구함 설치 조건

도구함은 다음 조건을 만족해야 설치됩니다.

- 메인 게임 WorldGuard 구역 안
- 파밍 구역 내부가 아님
- 파밍 구역 또는 파밍 노드에서 설정 거리 이상 떨어짐
- 다른 도구함에서 기본 35블럭 이상 떨어짐
- 도구함을 열 수 있는 통로가 최소 한 곳 존재

기본 건축 범위는 도구함 중심 기준 15×15입니다.

```yaml
building:
  claimRadius: 7
  minimumFarmDistance: 32.0
  minimumHouseDistance: 35.0
```

### 6-2. 건축 블럭

기본 건축 등급:

```text
나무 → 돌 → 철
```

일반 유저는 자신에게 권한이 있는 도구함 범위 안에서 전용 건축 블럭만 설치할 수 있습니다. 일반 마인크래프트 블럭은 게임 구역 안에서 건축용으로 사용할 수 없습니다.

### 6-3. 문

문 등급:

```text
나무 문 → 판금 문 → 강철 문
```

- 도구함의 현재 권한자와 등록 인원만 개폐 가능
- 권한 있는 유저는 정상적으로 문이 열림
- 권한 없는 유저가 연타하면 문은 열리지 않고 시야가 기본 180도 회전
- 기본 설정에서는 레드스톤으로 권한을 우회할 수 없음
- 문도 체력, 유지비, 수리, 업그레이드 대상

### 6-4. 건축 망치

망치를 주 손에 든 상태에서 사용합니다.

- 우클릭: 망치 모드 GUI 열기
- 건축물 좌클릭: 선택한 모드 실행

망치 모드:

| 모드 | 기능 |
|---|---|
| 수리 | 설정된 재료를 소모해 체력 회복 |
| 업그레이드 | 다음 건축 등급으로 변경 |
| 철거 | 건축물을 즉시 제거 |

철거 아이템 반환 여부와 업그레이드 비용은 설정 또는 관리자 패널에서 변경할 수 있습니다.

## 7. 도구함 권한, 유지비, 키패드

### 7-1. 권한 이전

- 키패드가 없는 도구함을 비인가 유저가 우클릭하면 권한이 즉시 그 유저에게 넘어갑니다.
- 권한 이전 시 이전 현재 권한자와 등록 인원은 초기화됩니다.
- 최초 설치자와 현재 권한자는 별도로 저장됩니다.
- 레이드 성공자가 권한을 얻어도 도구함 철거는 불가능합니다.
- 도구함 철거는 최초 설치자만 가능합니다.

### 7-2. 전체 권한 초기화

현재 권한자는 도구함 GUI의 `전체 권한 초기화` 버튼을 사용할 수 있습니다.

- 현재 권한자와 등록 인원이 모두 비활성화됨
- 도구함, 유지비, 키패드는 사라지지 않음
- 최초 설치자는 다시 우클릭해 권한을 되찾을 수 있음
- 키패드가 남아 있다면 비밀번호를 통과하거나 잠금을 파괴해야 함

### 7-3. 인원 관리

기본 최대 인원은 현재 권한자를 포함해 5명입니다. 도구함 GUI의 `인원 관리`에서 온라인 유저를 추가하거나 기존 인원을 제외할 수 있습니다.

### 7-4. 유지비

도구함의 `유지비 수납`에 필요한 재료를 넣습니다. 기본 4페이지이며 설정에서 1~4페이지로 바꿀 수 있습니다.

GUI 아이콘에 다음 정보가 표시됩니다.

- 다음 유지비 결제에 필요한 재료
- 현재 보관 중인 재료
- 24시간 유지에 필요한 총 재료
- 유지비 부족 또는 부식 중 경고

기본 유지비 결제 주기는 72,000틱, 즉 약 1시간입니다. 유지비가 부족한 등급의 건축물은 주기마다 체력이 감소하고 체력이 0이 되면 사라집니다.

### 7-5. 키패드 설치

현재 권한자가 도구함 GUI의 `키패드 설치/설정`을 누릅니다.

1. 숫자 4자리 입력
2. 확인 버튼 클릭
3. 최초 설치라면 기본 철 30개 차감

비밀번호는 평문이 아니라 솔트가 포함된 SHA-256 해시로 SQLite에 저장됩니다.

키패드가 있는 도구함을 비인가 유저가 우클릭하면 바로 권한이 넘어가지 않고 비밀번호 GUI가 열립니다.

- 정답: 도구함 권한 획득
- 오답: 설정된 피해를 받고 잠시 재입력 제한
- 레이드 피해로 키패드 체력이 0: 잠금 해제

키패드는 도구함 GUI의 `키패드 수리`에서 철을 사용해 수리할 수 있습니다.

## 8. 침낭

침낭 지급:

```text
/apoc give <플레이어> sleeping_bag
```

침낭은 기지 건축 범위 밖에도 설치할 수 있지만 다음 위치에는 설치할 수 없습니다.

- 파밍 WorldGuard 구역 경계에서 기본 30블럭 이내
- 자신에게 권한이 없는 다른 도구함에서 기본 30블럭 이내

침낭은 일반 블럭 파괴로 부술 수 없습니다. 설정된 도구로 반복해서 좌클릭해야 합니다.

기본 철거 횟수:

| 도구 | 필요 타격 수 |
|---|---:|
| 망치 | 3 |
| 나무 도끼 | 10 |
| 돌 도끼 | 7 |
| 철 도끼 | 5 |
| 다이아몬드 도끼 | 4 |
| 네더라이트 도끼 | 3 |

완전히 철거하면 기본적으로 침낭 아이템이 바닥에 떨어집니다. 침낭 위치와 설치자는 SQLite에 저장됩니다.

## 9. 파밍 플레이 방법

### 자원 블럭

1. 자원에 맞는 도구를 듭니다.
2. 자원 블럭을 좌클릭합니다.
3. 설정된 타격 횟수를 채우면 자원이 지급됩니다.
4. 블럭은 사라진 뒤 설정 시간이 지나면 리젠됩니다.

### 깡통

1. 설정된 근접 무기를 듭니다.
2. 깡통을 좌클릭합니다.
3. 보상을 받은 뒤 깡통은 리젠 대기 상태가 됩니다.

### 상자

1. 상자를 우클릭합니다.
2. 열린 보상 GUI에서 아이템을 가져갑니다.
3. 남은 아이템은 GUI를 닫을 때 인벤토리로 반환됩니다.

상자 블럭을 관리자가 철거하면 실제 컨테이너 안에 있던 아이템은 바닥에 떨어지고 노드 DB 기록이 삭제됩니다. 전역 보상 풀 전체가 복제되어 떨어지지는 않습니다.

## 10. 레이드 무기

### 가방 폭탄

- 가방 폭탄을 들고 블럭 우클릭
- 블럭 표면에 부착
- 설정된 시간이 지나면 폭발

### C4

- C4를 들고 블럭 우클릭
- 가방 폭탄보다 높은 기본 피해
- 폭발 전 1초마다 카운트 소리 재생

### 로켓 런처

- 주 손에 런처를 들면 액션바에 장전 탄약 표시
- 공중 또는 블럭 방향 우클릭: 로켓 발사
- 블럭을 가까이 바라볼 필요 없음
- 버리기 키 `Q`: 재장전
- 중력과 상승 보정이 적용된 포물선 비행
- 벽이나 엔티티에 충돌하면 폭발
- 최대 비행 시간이 지나면 설정에 따라 폭발 또는 제거

재장전 중 다음 행동을 하면 즉시 취소됩니다.

- 다른 핫바 슬롯 선택
- 숫자키로 주 손 아이템 교체
- 보조 손과 교환
- 인벤토리에서 런처 슬롯 변경
- 인벤토리 드래그

취소된 이전 재장전 타이머가 나중에 탄약을 채우지는 않습니다.

---

## 11. 관리자 강제 건축 모드

```text
/apoc admin build on
```

용도:

- 게임 구역과 파밍 구역 맵 꾸미기
- 파밍 시설 배치
- 갈갈이 설치
- 일반 블럭 강제 설치

주의사항:

- 강제 설치한 일반 블럭은 도구함 체력·유지비 데이터에 등록되지 않습니다.
- 등록된 도구함과 건축물은 DB 불일치를 막기 위해 강제로 부술 수 없습니다.
- 로그아웃하면 강제 건축 모드는 자동 해제됩니다.
- 작업이 끝나면 직접 끄는 것을 권장합니다.

```text
/apoc admin build off
```

## 13. 데이터 백업

서버를 종료한 상태에서 다음 파일을 함께 백업합니다.

```text
plugins/TGG-Apocalipse/config.yml
plugins/TGG-Apocalipse/apocalipse.db
```

`apocalipse.db`에는 다음 데이터가 저장됩니다.

- 도구함과 최초 설치자·현재 권한자
- 도구함 등록 인원과 유지비 아이템
- 키패드 비밀번호 해시와 체력
- 건축물 등급과 체력
- 파밍 노드와 리젠 상태
- 상자 보상
- 갈갈이 레시피
- 침낭 위치와 설치자

DB 파일은 서버 실행 중 직접 수정하지 않는 것을 권장합니다.

## 14. 문제 해결

### 플러그인이 켜지지 않음

- Java 21인지 확인
- Spigot 1.21.1인지 확인
- WorldEdit과 WorldGuard가 먼저 정상 로드됐는지 확인
- 일반 JAR가 아니라 Shadow JAR인지 확인

### 게임 구역에서 기능이 작동하지 않음

- WorldGuard 구역 ID와 `gameRegions` 철자가 같은지 확인
- `/apoc reload` 실행
- 대소문자는 자동 정규화되지만 공백과 오타는 확인

### 파밍 노드를 등록할 수 없음

- 대상 블럭을 6블럭 이내에서 바라보고 있는지 확인
- `farming.worldGuardRegions`를 설정했다면 대상이 파밍 구역 안인지 확인
- 자원 종류 또는 상자 등급이 `config.yml`에 존재하는지 확인

### 갈갈이를 설치할 수 없음

- 파밍 WorldGuard 구역 안인지 확인
- 관리자 강제 건축 모드가 켜져 있는지 확인
- 일반 갈갈이 아이템이 아니라 `/apoc give`로 받은 전용 아이템인지 확인

### 건축 블럭을 설치할 수 없음

- 메인 게임 구역 안인지 확인
- 파밍 구역이 아닌지 확인
- 자신에게 도구함 권한이 있는지 확인
- 도구함의 15×15 범위 안인지 확인
- 도구함 마지막 통로를 막고 있지 않은지 확인

### 도구함 권한이 바로 넘어가지 않음

- 키패드가 설치된 도구함은 비밀번호를 먼저 입력해야 함
- 키패드 체력을 레이드로 모두 깎으면 잠금이 해제됨

### 로켓이 발사되지 않음

- 런처에 장전된 탄약이 있는지 확인
- 재장전 중인지 확인
- 발사 쿨타임 중인지 확인
- 탄약이 없다면 로켓을 인벤토리에 넣고 `Q`로 재장전

### 설정 파일을 잘못 수정함

- 서버 콘솔에서 리로드 실패 메시지 확인
- YAML 들여쓰기는 탭이 아닌 공백 사용
- 문제가 생기기 전 `config.yml` 백업본 복원
